## F-01 · 역할 기반 온보딩 플로우
**우선순위:** HIGH

### 개요
신규 가입 또는 첫 로그인 사용자에게 역할·직군·지역을 3단계로 선택하게 하여, 각자에게 최적화된 홈 피드를 자동으로 구성한다.
"들어오면 뭘 눌러야 할지 모르겠다"는 입문자 허들을 온보딩 단계에서 차단하는 것이 핵심 목적이다.

### 연결 퍼소나
- **Persona 1 · 박지훈** — 입문자, 역할·카테고리 이해 어려움
- **Persona 2 · 이수연** — 직군 맞춤 피드 빠른 진입 필요

### 입력 · 출력

| 항목 | 내용 |
|---|---|
| 진입 조건 | 신규 가입 완료 직후 / 기존 회원 최초 앱 업데이트 후 첫 로그인 |
| 입력값 | 역할 (복수 선택 허용) · 관심 직군 (복수 선택) · 활동 지역 (단일/전국) |
| 출력값 | 역할·직군·지역 기반으로 필터링된 홈 피드 구성 + 마이페이지 프로필 초기값 저장 |
| 건너뛰기 | 각 단계에서 "건너뛰기" 버튼 제공 → 전국/전체 직군 기본값 적용 |

### 화면 플로우 (3 Step)

**Step 1 · 역할 선택** `[필수]`
- "찍는 사람(스태프·감독)", "나오는 사람(배우·모델)", "지금 알아보는 중(입문자)" 3가지 옵션
- 복수 선택 가능 — 겸업자를 위해 2개 이상 선택 시 통합 피드 구성
- 건너뛰기 시 전체 기본값 적용

**Step 2 · 관심 직군 선택** `[선택]`
- 역할에 따라 직군 목록 다이나믹 노출
  - 스태프 선택 시: 연출부 · 촬영부 · 조명부 · 편집·색보정 · 사운드 · 제작부 등
  - 배우 선택 시: 배우(연기) · 모델 · 나레이션 등
- 최대 3개 선택 권장, 제한은 없음
- 건너뛰기 시 전체 직군 노출

**Step 3 · 활동 지역 선택** `[선택]`
- 시·도 단위 선택 (서울 · 경기 · 부산 · 대구 · 전국 등)
- "전국" 선택 시 지역 필터 미적용
- 이후 홈 퀵 필터 바에서 언제든 변경 가능

### 예외 처리

| 케이스 | 처리 방식 |
|---|---|
| 건너뛰기 남용 | 온보딩 건너뛴 사용자가 첫 공고 탐색 후 30초 이상 체류 시, 하단 넛지 토스트 표시 — "역할을 설정하면 맞춤 공고를 볼 수 있어요" |
| 역할 오선택 | 마이페이지 → "내 역할·직군 수정" 메뉴 상시 노출. 변경 즉시 홈 피드 재구성. |
| 네트워크 오류 | 온보딩 데이터 로컬 임시 저장 후 재시도. 저장 실패 시 다음 진입 때 온보딩 재진행 유도. |

### 인터뷰 근거
> "업계 전문용어를 모르는 초보 사용자가 이용하기엔 진입장벽이 높다" — SeungYoon Hong

> "사이트를 새로 접하거나 오랜만에 접하는 사람한테 보기 어려운 구성이다" — SeungYoon Hong

## F-02 · 공고 카드 UI
**우선순위:** HIGH

### 개요
공고 리스트에서 상세 페이지 진입 없이 핵심 조건을 한눈에 스캔할 수 있는 카드형 UI.
"하나하나 다 눌러봐야 아는 불편함"을 리스트 단계에서 해소하는 것이 목표다.
구인자 측 공고 작성 시 핵심 필드를 필수 입력으로 강제하여 카드 정보의 품질을 보장한다.

### 연결 퍼소나
- **Persona 1 · 박지훈** — 조건 확인에 소요 시간 최소화 필요
- **Persona 2 · 이수연** — 30초 내 지원 여부 판단 목표

### 카드 노출 정보 명세

| 필드명 | 표시 위치 | 입력 주체 | 필수 여부 |
|---|---|---|---|
| 공고 제목 | 카드 상단 (Bold) | 구인자 | 필수 |
| 역할 태그 | 제목 하단 칩 | 구인자 (선택형) | 필수 |
| 촬영 지역 | 메타 정보 행 | 구인자 | 필수 |
| 촬영 기간 | 메타 정보 행 | 구인자 | 필수 |
| 페이 여부 | 우측 뱃지 | 구인자 (유료/무보수/협의) | 필수 |
| 마감일 | 우측 하단 | 구인자 | 필수 |
| 경력 무관 여부 | 하단 초록 뱃지 | 구인자 (토글) | 선택 |

### 상태 뱃지 규칙

- **마감 임박 뱃지 (빨강):** 마감일까지 3일 이하 남은 공고에 자동 표시. "D-3", "D-1" 형식.
- **경력 무관 뱃지 (초록):** 구인자가 "경력 무관" 토글 ON 시 카드에 뱃지 자동 표시.
- **페이 표시 (색상 구분):** 유료 → 파란 텍스트, 무보수 → 회색 텍스트, 협의 → 주황 텍스트.

### 구인자 측 필수 입력 강제 정책

| 필드 | 입력 형태 | 미입력 시 처리 |
|---|---|---|
| 역할 태그 | 칩 선택 (다중) | 공고 등록 버튼 비활성화 |
| 촬영 지역 | 드롭다운 (시·도) | 공고 등록 버튼 비활성화 |
| 촬영 기간 | 날짜 범위 피커 | 공고 등록 버튼 비활성화 |
| 페이 여부 | 라디오 (유료/무보수/협의) | 공고 등록 버튼 비활성화 |
| 마감일 | 날짜 피커 | 공고 등록 버튼 비활성화 |

### 예외 처리

| 케이스 | 처리 방식 |
|---|---|
| 구인자 입력 부담 | 필수 필드는 모두 선택형(칩·토글·드롭다운)으로 제공. 자유 텍스트 입력 최소화하여 작성 시간 30초 이내 목표. |
| 기존 공고 마이그레이션 | 기존 공고 중 필수 필드 미입력 건은 "정보 미기재" 뱃지 표시. 구인자에게 수정 유도 알림 발송. |
| 이미지 없는 공고 | 직군에 따라 기본 일러스트 자동 적용 (역할 아이콘 기반). |

### 인터뷰 근거
> "상세 페이지에 들어가야만 성명, 나이제한 등을 알 수 있어서 하나하나 다 눌러봐야 하는 불편함" — SeungYoon Hong

> "첫 화면에 8~9번 스크롤해야 보이는 정보들이 있어서 한 눈에 알아보기 쉽게 바뀌면 좋겠다" — 김가루

> "촬영 기간 등 특별한 이유 없으면 들어가지 않음" — 김가루

## F-03 · 홈 퀵 필터 바
**우선순위:** MID

### 개요
홈 화면 상단에 고정된 원탭 필터 칩 모음.
"매일 전체 리스트를 수동으로 훑는" 헤비유저의 반복 비효율을 줄이고,
지방 거주 입문자가 자신에게 해당하는 공고를 즉시 추려볼 수 있도록 한다.
필터 과부하 방지를 위해 기본 노출은 4개 이하로 제한한다.

### 연결 퍼소나
- **Persona 1 · 박지훈** — "내 지역" 필터로 지방 공고 즉시 추출
- **Persona 2 · 이수연** — 직군·마감 필터로 매일 탐색 시간 단축

### 기본 노출 필터 항목 (4개)

| 필터명 | 동작 | 기본값 |
|---|---|---|
| 내 지역 | 온보딩에서 설정한 지역 기준 자동 적용 / 미설정 시 현재 위치 제안 | OFF |
| 이번 주 마감 | 마감일이 오늘 기준 7일 이내인 공고만 노출 | OFF |
| 경력 무관 | "경력 무관" 뱃지가 있는 공고만 노출 | OFF |
| 무보수 제외 | 페이 여부가 "무보수"인 공고 제외 | OFF |

### 확장 필터 항목 ("더 보기" 클릭 시)

| 필터명 | 입력 형태 |
|---|---|
| 직군 선택 | 칩 다중 선택 (온보딩 직군값 기본 적용) |
| 페이 범위 | 라디오 (유료 / 무보수 / 협의 / 전체) |
| 지역 직접 선택 | 시·도 드롭다운 |
| 촬영 기간 | 날짜 범위 피커 |

### 필터 동작 규칙

1. **복수 필터 AND 조합** — 여러 필터를 동시에 ON 하면 모든 조건을 만족하는 공고만 노출 (교집합)
2. **필터 적용 상태 고정** — 앱을 닫아도 마지막 필터 설정값 유지. 다음 진입 시 동일 필터 자동 적용 `[로컬 저장 필요]`
3. **결과 0건 처리** — 필터 조합 결과가 0건일 때 "해당 조건의 공고가 없어요. 필터를 조정해보세요" 안내 + 필터 초기화 버튼 제공

### 예외 처리

| 케이스 | 처리 방식 |
|---|---|
| 필터 과부하 | 기본 노출 4개 고정. 추가 필터는 "더 보기" 뒤에 격리. 동시 활성 필터 수 상단에 뱃지로 표시 (예: "필터 3개 적용 중"). |
| 온보딩 미완료 사용자 | "내 지역" 필터 탭 시 "지역을 설정하면 내 주변 공고를 볼 수 있어요" 바텀시트로 온보딩 지역 설정 유도. |
| 위치 권한 거부 | 현재 위치 기반 지역 제안 불가 시 드롭다운 직접 선택 화면으로 폴백. |

> "매일 자기 전에 습관적으로 들어가서 스태프 공고를 확인한다" — 김가루
