# 필름메이커스 UX/UI 개선 전략 v1
**작성일:** 2026-04-24
**대상 서비스:** 필름메이커스
**핵심 타겟:** 미디어 업계 종사 20대 대학생 ~ 30대 취준생
**경쟁사:** 미디어잡, 디렉터스
**리서치 방법:** 경쟁사 실사용자 10명 인터뷰 + 어피니티 다이어그램

---

## 1. 주요 페인포인트 상위 3가지

### 페인포인트 1 — UI 노후화 및 전반적 가독성 저하

**발언 근거:**
- "노후화된 인터페이스. 보기 어렵다." (김가루)
- "화면 구성이 복잡하여 전반적인 가독성이 떨어지므로 원하는 공고를 찾는데 오래 걸린다." (SeungYoon Hong)
- "모든 게시판에 버튼이 꽉 차있어서 답답하게 보인다 (시각적인 불편함)." (SeungYoon Hong)
- "가독성이 떨어지는 색감 때문에 답답하고 불편하다. (색이 어두움)" (SeungYoon Hong)
- "커뮤니티에 가까운 형태라 UI가 예쁘지 않음." (SeungYoon Hong)

**근본 원인:**
서비스가 커뮤니티 게시판 형태로 출발해 구인구직 플랫폼으로 성장했지만, UI는 초기 커뮤니티 구조에 머물러 있음. 정보 밀도가 높고 시각적 위계가 없는 레이아웃이 사용자의 인지 부하를 과도하게 높임. 특히 신규 유저 또는 오랜만에 접속하는 유저에게 진입장벽이 매우 높게 작용함.

---

### 페인포인트 2 — 정보 탐색 구조의 비직관성 (IA 문제)

**발언 근거:**
- "상세 페이지에 들어가야만 성명, 나이제한 등을 알 수 있어서 하나하나 다 눌러봐야 하는 불편함." (SeungYoon Hong)
- "홈 화면에 기본적인 검색창이 없고 게시판 내부 검색기능도 복잡하게 되어있다." (SeungYoon Hong)
- "첫 화면에 8~9번 스크롤해야 보이는 정보들이 있어서 한 눈에 알아보기 쉽게 바뀌면 좋겠다." (김가루)
- "'만드는 사람', '나오는 사람' 이런 식으로 되어있어서 구별하기 어려움." (김가루)
- "카테고리 분류가 정신없이 너무 많음." (김가루)
- "배우 프로필과 구인구직 게시판 구분이 모호해서 헷갈리게 되어있다." (SeungYoon Hong)
- "업계 전문용어를 모르는 초보 사용자가 이용하기엔 진입장벽이 높다." (SeungYoon Hong)

**근본 원인:**
게시판 구조가 업계 내부 관행 언어 기반으로 설계되어 있어, 입문자나 비정기 사용자가 원하는 정보에 도달하는 경로가 불명확함. 공고 리스트에서 핵심 조건(나이, 기간, 역할 등)을 미리 볼 수 없어 '열어봐야 안다'는 비효율이 반복 발생.

---

### 페인포인트 3 — 지역 편중 및 커뮤니티 기능 실질적 비활성화

**발언 근거:**
- "공고가 서울에만 몰려있다." (JIHWAN)
- "대부분 공고가 서울로 몰려있어서 지방 거주자가 불편함을 느낀다." (u3in)
- "커뮤니티 게시판에 질문을 올려도 답변이 거의 달리지 않아서 소통이 약화되어있다." (u3in)
- "대체제가 없어서 불편함을 감수하고서 사용하게 됨." (u3in)
- "국내에서 중단편 독립 영화 배우를 구할 수 있는 유일한 커뮤니티이다." (u3in)

**근본 원인:**
플랫폼 내 콘텐츠(공고)가 수도권 사용자·구인자 중심으로 누적되어 지방 사용자가 실질적 혜택을 받지 못하는 네트워크 불균형 문제. 커뮤니티 기능은 존재하지만 참여 유도 메커니즘이 없어 질문-답변 루프가 작동하지 않음. "대체재가 없어서 쓴다"는 발언은 충성도가 아닌 Lock-in에 의한 이용임을 시사 — 경쟁사 등장 시 이탈 위험 높음.

## 2. 차별화 전략 방향 3가지

### 전략 1 — 역할 기반 개인화 온보딩 + UI 현대화

**해결하는 페인포인트:** #1 (UI 노후화), #2 (비직관적 IA)

**전략 방향:**
첫 진입 시 사용자 역할(감독/스태프/배우/입문자 등)을 선택하게 하여, 각 역할에 최적화된 홈화면과 메뉴 구조를 보여주는 역할 맞춤형 UX 설계. 동시에 전체적인 색감, 여백, 타이포그래피를 현대적으로 리디자인.

**기능/경험 아이디어:**
- **온보딩 플로우:** 앱 첫 진입 시 "나는 찍는 사람 / 나오는 사람 / 지금 알아보는 중" 3단계 선택 → 역할별 맞춤 홈 구성 제공
- **공고 카드 UI:** 리스트에서 촬영 기간, 역할, 지역, 페이 여부를 한눈에 확인 가능한 카드형 레이아웃 (상세 진입 없이 핵심 조건 스캔 가능)

**리스크 및 주의점:**
- 기존 사용자 이탈: UI를 급격히 바꾸면 오랜 사용자가 거부감을 느낄 수 있음. 헤비유저층 이탈 주의.
- 역할 선택의 한계: 미디어 업계는 한 사람이 감독+스태프+배우를 겸하는 경우가 많아, 단일 역할 분류가 오히려 불편을 유발할 수 있음. 복수 역할 선택 + 언제든 전환 가능 구조 필수.

---

### 전략 2 — 스마트 필터 + 홈 검색창 중심의 탐색 경험 재설계

**해결하는 페인포인트:** #2 (탐색 비효율), #3 (지역 편중)

**전략 방향:**
홈화면 최상단에 통합 검색창을 배치하고, 지역·직군·페이 유무·촬영 기간 등의 조합 필터를 직관적으로 제공. 검색 결과에 지역 공고 가중치 알고리즘을 도입해 비수도권 사용자 경험 개선.

**기능/경험 아이디어:**
- **홈 퀵 필터 바:** 상단 고정 칩 형태로 "내 지역", "이번 주 마감", "무보수 제외" 등 원탭 필터 제공
- **AI 맞춤 추천:** 사용자의 열람·지원 이력 기반으로 "당신과 비슷한 사람이 본 공고" 섹션 노출 (초기엔 직군+지역 기반 룰 기반으로 시작)

**리스크 및 주의점:**
- 콜드스타트 문제: AI 추천은 데이터가 충분히 쌓이기 전까지 품질이 낮아 오히려 신뢰를 떨어뜨릴 수 있음. 초기엔 룰 기반 필터를 명확히 보여주는 것이 더 신뢰감을 줌.
- 필터 과부하: 필터 옵션을 너무 많이 제공하면 탐색이 오히려 복잡해짐. 자주 쓰는 조건 3~4개만 우선 노출하고 나머지는 "더 보기"로 숨기는 점진적 노출 전략 필요.

---

### 전략 3 (수정) — 커뮤니티를 "광장"이 아닌 "연결 구조"로 재설계

**해결하는 페인포인트:** #3 (커뮤니티 비활성), #1 (전문용어 진입장벽), #2 (정보 불투명성)

**핵심 방향 전환:**
"시니어가 커뮤니티에 참여하게 만든다" → "시니어의 구인 행위 자체가 입문자에게 도움이 되는 구조를 만든다"

**전략 배경 — 구조적 문제 인식:**
입문자는 질문할 동기가 있지만 시니어는 답변할 동기가 없는 비대칭 구조가 핵심 문제. 시니어는 취업 후 플랫폼 이용 빈도가 낮아지고("취업을 하면 잘 안하게 된다"), 익명 커뮤니티에서 전문성을 소모할 이유가 없음. 따라서 시니어를 "커뮤니티 참여자"가 아닌 "서비스 수혜자"로 설계해야 함.

**기능/경험 아이디어:**
- **공고 기반 Q&A 연동:** 공고 상세 페이지에 "이 공고에 대해 질문하기" 버튼 추가. 구인자(시니어)가 답변하면 해당 공고 상단 노출 혜택 제공 → 시니어는 공고 올릴 때 한 번만 참여해도 구조가 돌아감.
- **공고 투명성 강화:** 공고 작성 시 역할 설명 / 촬영 기간 / 페이 여부 / 경력 무관 여부를 필수 입력 필드로 구조화 → 입문자가 궁금한 것 대부분이 공고 자체에서 해결되어 Q&A 수요 자체를 줄임.
- **업계 용어 자동 툴팁:** 공고 및 게시판 내 전문용어(DIT, 오퍼레이터, 믹싱 등)에 자동 툴팁 표시 → 시니어 개입 없이도 입문자 허들을 낮추는 콘텐츠 인프라 방식.

**리스크 및 주의점:**
- 구인자 입력 부담: 공고 필수 필드를 늘리면 구인자가 귀찮아서 공고 자체를 안 올릴 수 있음. 필드 수를 최소화하고 원탭 선택형(토글/칩)으로 설계 필수.
- 잔여 Q&A 수요: 공고 투명성을 높여도 "이 업계는 어떻게 시작하나요?" 같은 커리어 질문은 남음. FAQ 아카이브 형태로 쌓아두고 유사 질문은 기존 답변으로 자동 연결하는 구조가 현실적.

## 3. 종합 인사이트

유저들은 필름메이커스를 "대체재가 없어서 쓰는 플랫폼"으로 인식하고 있음. 이는 콘텐츠 자산(정보량)은 있지만 경험 자산(UX)이 매우 취약하다는 의미. 경쟁사가 UX를 갖추기 전에 빠르게 경험을 개선하는 것이 방어 전략이자 성장 전략임.

| 전략 | 핵심 해결 문제 | 우선순위 근거 |
|---|---|---|
| 전략 1: 역할 기반 온보딩 + UI 현대화 | UI 노후화, 비직관적 IA | 이탈 방지 및 신규 유저 유입의 기본 조건 |
| 전략 2: 스마트 필터 + 탐색 재설계 | 탐색 비효율, 지역 편중 | 핵심 사용 행동(공고 탐색) 직접 개선 |
| 전략 3: 공고 연동형 커뮤니티 구조 | 커뮤니티 비활성, 진입장벽 | Lock-in 의존 탈피 및 장기 차별화 |
